# Subagent: rank-signature modeler
**Model:** Opus-class subagent · **Spawned by:** the main analysis session

## Instructions given
Build a predictor of what separates Silver player-games from Iron player-games, using match + timeline data from the Riot API, then apply it to djfalcon's ranked games to identify which levers to turn. Statistical requirements set by the thread owner (a statistician): (1) rank as the label (exogenous to a single game), not win/loss, to dodge outcome circularity; (2) two feature families — controllable behavior inputs (phase-windowed CS, early deaths, ward cadence/timing, item timing) vs outcome-contaminated descriptors (objectives, gold share, KDA) — model levers from the first family only, and report the discrimination gap between families as its own finding; (3) fixed-window features (stats@10/@14) not per-game totals, with a discrete-time hazard treatment of time-to-first-death as the survival component; (4) match-level clustering respected everywhere (GroupKFold CV by match, cluster bootstrap CIs) — no pseudo-replication; (5) Bronze as a never-trained ordinal sanity check (should score between Iron and Silver); (6) apply to djfalcon per-game with feature attribution, aggregated into a lever ranking with uncertainty. Simple models (regularized logistic, gradient boosting) are fine; the assumptions must be smart and stated. Iterate publicly: post short findings comments on the subtask issue as work progresses, then a final report comment.
